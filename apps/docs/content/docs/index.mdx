Panqueue is a job queue for JavaScript and TypeScript, backed by Redis. It is small,
typed end to end, and focused on the two things a queue library has to nail:
**speed** and **developer experience**.

Define your queues once in a shared config, get full type safety on every `enqueue`
and every worker handler, and let Panqueue handle the parts you don't want to write
yourself: retries, dead-lettering, crashed-worker recovery, and clean shutdowns.

<Callout type="warn" title="Pre-release">
  Panqueue is not released yet. The docs describe the current shape of the library while the API is
  stabilising toward the first public version.
</Callout>

## What you get

- **Type-safe queues and payloads** from a single shared config
- **A small producer API** for enqueueing jobs from anywhere in your app
- **Inline worker handlers** that stay separate from worker lifecycle
- **Automatic retries** with a bounded dead-letter set, no setup required
- **Recovery from crashed workers** so jobs are never lost in the void
- **Clean shutdowns** that hand in-flight work back to the queue for someone else

## Runtime support

Panqueue currently targets **Node, Bun, and Deno** with packages published to both
**npm** and **JSR** from a single TypeScript source tree. All you need beyond
Panqueue is a Redis 7+ server.

Detailed API reference, deployment notes, and operational guides will come after
the first tutorial pages are stable.
