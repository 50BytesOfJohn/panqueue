Panqueue is a Deno-first queue library backed by Redis.

The v0.1 API is built around a small set of ownership boundaries:

- `definePanqueueConfig` describes Redis connection config and typed queues.
- `QueueClient` owns the producer Redis connection and enqueues jobs.
- `defineWorker` creates pure worker definitions with typed processors.
- `WorkerPool` owns the worker-side Redis connections and runs registered
  worker definitions.

Users pass Redis connection configuration, not Redis client objects.

## Example

```ts
import { definePanqueueConfig } from "@panqueue/config";
import { createQueueClient } from "@panqueue/client";
import { defineWorker, WorkerPool } from "@panqueue/worker";

type Queues = {
  email: { to: string; subject: string; body: string };
};

export const pq = definePanqueueConfig<Queues>({
  redis: { url: Deno.env.get("REDIS_URL")! },
  queues: {
    email: { mode: "global" },
  },
});

const client = createQueueClient(pq);
await client.enqueue("email", {
  to: "hello@example.com",
  subject: "Welcome",
  body: "Thanks for trying Panqueue.",
});

const emailWorker = defineWorker(pq, "email", async (job) => {
  await sendEmail(job.data);
});

const pool = new WorkerPool(pq);
pool.register(emailWorker);
await pool.start();
```
